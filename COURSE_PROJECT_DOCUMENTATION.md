# КУРСОВОЙ ПРОЕКТ
## «Платформа для организации мероприятий»

---

## АННОТАЦИЯ

Курсовой проект на тему «Платформа для организации мероприятий» состоит из программного средства, пояснительной записки и графической части. Программное средство предназначено для комплексной организации и управления мероприятиями, включая функционал бронирования мест, управления заказами и платежами, генерации QR-кодов, системы отзывов и уведомлений, а также управления пользователями.

Программное средство разработано в среде Visual Studio Code с использованием языка C# и современных технологий: React.js для клиентской части, ASP.NET Core MVC для серверной логики и Microsoft SQL Server для надежного хранения данных. Проект включает полный цикл разработки: создание современного пользовательского интерфейса, реализацию бизнес-логики, работу с реляционной базой данных, систему аутентификации на основе JWT-токенов и RESTful API для взаимодействия с клиентской частью.

Пояснительная записка выполнена на 38 листах, содержит четыре раздела и два приложения.

Графическая часть выполнена на двух листах, включает диаграмму вариантов использования и диаграмму последовательности.

## ВВЕДЕНИЕ

В современном мире цифровизация охватывает все сферы жизни, включая организацию и проведение мероприятий. Традиционные методы бронирования билетов через телефонные звонки и очереди в кассах уступают место современным электронным платформам, которые обеспечивают удобство, прозрачность и эффективность процесса продажи билетов.

Данный курсовой проект представляет собой разработку «Платформа для организации мероприятий» — комплексного веб-приложения, предназначенного для автоматизации процессов организации мероприятий, управления бронированием мест, обработки заказов и платежей, а также взаимодействия между организаторами и посетителями мероприятий.

Веб-приложение представляет собой программное обеспечение, работающее в браузере и предоставляющее пользователям доступ к функционалу через интуитивно понятный интерфейс веб-страницы. Приложение построено по архитектуре клиент-сервер, где клиентская часть обеспечивает взаимодействие с пользователем, а серверная часть обрабатывает бизнес-логику и управляет данными.

Актуальность темы обусловлена несколькими факторами:
- **Растущая потребность в цифровизации**: Организаторы мероприятий все чаще переходят на электронные системы бронирования для повышения эффективности и снижения операционных затрат.
- **Удобство для пользователей**: Современные пользователи ожидают возможность бронирования билетов онлайн в любое время и с любого устройства.
- **Прозрачность и контроль**: Электронные системы обеспечивают полную прозрачность процесса бронирования, возможность отслеживания заказов и управления билетами.
- **Масштабируемость**: Цифровые платформы позволяют легко масштабировать бизнес и обрабатывать большое количество заказов одновременно.

Цели курсового проектирования:
- закрепить, углубить и систематизировать теоретические знания и практические навыки, полученные в ходе изучения учебного предмета;
- сформировать навыки разработки алгоритмов решения задач и проектирования архитектуры программных систем;
- освоить современные технологии веб-разработки: React.js, ASP.NET Core, Entity Framework Core;
- разработать полнофункциональное программное средство, провести его отладку, тестирование и верификацию;
- разработать удобный и современный интерфейс пользователя, систему навигации и управления;
- освоить работу с реляционными базами данных и проектирование схем данных;
- реализовать систему безопасности и аутентификации пользователей.

Пояснительная записка содержит следующие разделы:

1. **«Назначение и область применения»** — описывается назначение программы, краткая характеристика области применения, анализ предметной области и инфологическое проектирование разрабатываемой базы данных. ## 1. НАЗНАЧЕНИЕ И ОБЛАСТЬ ПРИМЕНЕНИЯ

### 1.1. Назначение программы

Разработанное веб-приложение представляет собой комплексную платформу для автоматизации процессов организации мероприятий и управления бронированием мест. Программа обеспечивает полный цикл работы с мероприятиями: от создания события организатором до покупки билетов посетителями, обработки платежей и управления заказами.

Основные возможности программы:
- **Управление мероприятиями**: Создание, редактирование и удаление мероприятий с указанием детальной информации (дата, время, место, описание, состав исполнителей).
- **Управление местами**: Создание схемы зала с различными типами мест (стандартные, VIP, для инвалидов), управление доступностью мест и их резервацией.
- **Система бронирования**: Интерактивная схема зала для визуального выбора мест, резервация мест на определенное время, автоматическое освобождение неоплаченных мест.
- **Управление заказами**: Группировка билетов в заказы, отслеживание статуса заказа, управление историей покупок пользователя.
- **Обработка платежей**: Интеграция с платежными системами, отслеживание статуса платежей, поддержка различных способов оплаты.
- **Генерация билетов**: Автоматическая генерация QR-кодов для каждого билета, возможность проверки билетов при входе на мероприятие.
- **Система отзывов**: Возможность для посетителей оставлять отзывы и оценки мероприятиям, модерация отзывов администраторами.
- **Уведомления**: Система уведомлений для пользователей о новых событиях, изменениях в билетах, напоминаниях о предстоящих мероприятиях.
- **Обратная связь**: Форма обратной связи для связи пользователей с администрацией, отслеживание статуса обращений.

### 1.2. Область применения

Область применения программы охватывает широкий спектр организаций и пользователей, связанных с проведением мероприятий:

**Организаторы мероприятий:**
- Концертные залы и площадки
- Театры и культурные центры
- Спортивные арены и стадионы
- Образовательные учреждения
- Корпоративные организаторы событий
- Частные организаторы мероприятий

**Администраторы системы:**
- Сотрудники организаций, управляющие платформой
- Системные администраторы
- Модераторы контента и отзывов

**Посетители мероприятий:**
- Физические лица, желающие посетить мероприятия
- Корпоративные клиенты
- Группы и коллективы

### 1.3. Группы пользователей

Система предусматривает три основные группы пользователей с различными правами доступа:

**1. Администраторы системы:**
- Полный доступ к управлению системой
- Создание и редактирование мероприятий
- Управление пользователями
- Модерация отзывов
- Обработка обращений через форму обратной связи
- Просмотр аналитики и отчетности
- Управление схемами мест для мероприятий

**2. Организаторы мероприятий:**
- Создание новых мероприятий с указанием даты, времени и места
- Управление схемой мест для мероприятия
- Редактирование информации о мероприятии
- Просмотр списка забронированных мест
- Управление типами билетов
- Просмотр статистики по мероприятиям

**3. Посетители (обычные пользователи):**
- Регистрация и авторизация в системе
- Просмотр списка доступных мероприятий с фильтрацией и поиском
- Бронирование мест через интерактивную схему зала
- Покупка билетов и оформление заказов
- Просмотр истории покупок и билетов
- Управление профилем пользователя
- Оставление отзывов о посещенных мероприятиях
- Получение уведомлений о новых событиях и изменениях

### 1.4. Анализ существующих решений

Для проработки функционала проектируемого приложения были рассмотрены следующие аналоги на рынке:

Достоинства:

- Удобная система оплаты с поддержкой различных платежных методов
- Интеграция с различными площадками и организаторами
- Мобильное приложение для удобства пользователей

Недостатки:

- Сложный процесс добавления новых мероприятий для небольших организаторов
- Ограниченные возможности кастомизации интерфейса

**Ticketpro** (https://www.ticketpro.by/)

Достоинства:
- Профессиональные инструменты для организаторов мероприятий
- Развитая система аналитики и отчетности
- Поддержка различных типов билетов и схем мест
- Интеграция с системами контроля доступа
- Высокий уровень безопасности и надежности

Недостатки:
- Высокая стоимость использования для небольших организаторов
- Сложный интерфейс для простых пользователей
- Избыточный функционал для небольших мероприятий
- Длительный процесс настройки и интеграции

**Отличия разработанного решения:**

Разработанное решение ориентировано на небольшие и средние мероприятия, предлагая:

1. **Простота использования**: Интуитивно понятный интерфейс как для организаторов, так и для посетителей, минимальная кривая обучения.

2. **Гибкость настройки**: Возможность настройки под конкретные потребности организатора, кастомизация схем мест и типов билетов.

3. **Отсутствие сложных интеграций**: Самостоятельное решение, не требующее интеграции с множеством внешних сервисов, что упрощает развертывание и поддержку.

4. **Экономичность**: Отсутствие комиссий и скрытых платежей, возможность использования на собственной инфраструктуре.

5. **Полный контроль**: Организатор имеет полный контроль над данными, мероприятиями и пользователями.

6. **Современные технологии**: Использование актуальных технологий обеспечивает высокую производительность и возможность дальнейшего развития.

### 1.5. Системные требования

**Минимальные системные требования для сервера:**
- Процессор: 2.0 GHz (рекомендуется 4 ядра)
- Оперативная память: 8 GB (рекомендуется 16 GB)
- Свободное место на диске: 10 GB (для базы данных и приложения)
- Операционная система: Windows Server 2016+, Linux (Ubuntu 20.04+), macOS 10.15+
- .NET Runtime: .NET 8.0 или выше
- SQL Server: SQL Server 2019 Express или выше (или LocalDB для разработки)
- Браузер для администрирования: Chrome 90+, Firefox 88+, Edge 90+

**Минимальные системные требования для клиента:**
- Процессор: 1.6 GHz
- Оперативная память: 4 GB
- Свободное место на диске: 500 MB (для кэша браузера)
- Браузер: Chrome 80+, Firefox 75+, Safari 13+, Edge 80+
- Интернет-соединение: стабильное соединение со скоростью не менее 1 Мбит/с

**Программное обеспечение для разработки:**
- Node.js: 18.0+ (для клиентской части)
- .NET SDK: 8.0+ (для серверной части)
- SQL Server Management Studio: 18.0+ (для работы с базой данных)
- Visual Studio Code или Visual Studio 2022 (для разработки)

## 2. ТЕХНИЧЕСКИЕ ХАРАКТИСТИКИ

### 2.1. Постановка задачи

#### Функциональные требования к программе

**Для администратора системы:**
- Реализовать возможность создания и управления мероприятиями (добавление, редактирование, удаление)
- Обеспечить добавление и редактирование информации о местах проведения
- Предусмотреть управление пользователями системы (просмотр списка, блокировка, удаление)
- Реализовать модерацию отзывов пользователей (одобрение, отклонение, удаление)
- Обеспечить обработку обращений через форму обратной связи
- Предусмотреть просмотр аналитики: статистика по мероприятиям, продажам, пользователям
- Реализовать управление типами билетов и настройками системы

**Для организатора мероприятий:**
- Обеспечить создание новых мероприятий с указанием даты, времени, места, описания и состава исполнителей
- Реализовать управление схемой мест для мероприятия (создание, редактирование, удаление мест)
- Предусмотреть возможность редактирования информации о мероприятии до начала продаж
- Обеспечить просмотр списка забронированных и проданных мест
- Реализовать управление типами билетов для мероприятия
- Предусмотреть просмотр статистики по конкретному мероприятию

**Для посетителей:**
- Реализовать регистрацию и авторизацию в системе с подтверждением email
- Обеспечить просмотр списка доступных мероприятий с возможностью фильтрации по категориям, дате, цене
- Реализовать поиск мероприятий по названию, исполнителю, месту проведения
- Обеспечить возможность бронирования мест через интерактивную схему зала
- Реализовать оформление заказа с группировкой нескольких билетов
- Обеспечить обработку платежей с поддержкой различных способов оплаты
- Реализовать генерацию QR-кодов для билетов
- Предусмотреть просмотр истории покупок и активных билетов
- Обеспечить возможность оставления отзывов о посещенных мероприятиях
- Реализовать систему уведомлений о новых событиях, изменениях в билетах, напоминаниях
- Предусмотреть возможность связи с администрацией через форму обратной связи

#### Нефункциональные требования

- **Производительность**: Время отклика системы не должно превышать 2 секунд для стандартных операций
- **Безопасность**: Использование JWT-токенов для аутентификации, хеширование паролей (BCrypt), защита от SQL-инъекций
- **Масштабируемость**: Возможность обработки до 1000 одновременных пользователей
- **Надежность**: Доступность системы не менее 99%, автоматическое резервное копирование базы данных
- **Удобство использования**: Интуитивно понятный интерфейс, адаптивный дизайн для различных устройств
- **Совместимость**: Работа в основных современных браузерах (Chrome, Firefox, Safari, Edge)

### 2.2. Описание организации данных

#### Входные данные

Входными данными системы являются:

1. **Информация о мероприятиях:**
   - Название мероприятия
   - Дата и время проведения
   - Место проведения и адрес
   - Описание мероприятия
   - Категория и тип события
   - Состав исполнителей (lineup)
   - Изображение мероприятия
   - Ценовая информация

2. **Данные пользователей:**
   - Имя пользователя
   - Email адрес (уникальный)
   - Пароль (хранится в хешированном виде)
   - Статус верификации email
   - Токен верификации

3. **Схемы мест для мероприятий:**
   - Ряд и номер места
   - Тип места (стандартное, VIP, для инвалидов)
   - Цена места
   - Статус места (доступно, зарезервировано, продано)

4. **Данные о заказах:**
   - Уникальный номер заказа
   - Общая сумма заказа
   - Статус заказа

5. **Данные о платежах:**
   - Способ оплаты
   - Сумма платежа
   - ID транзакции от платежной системы
   - Статус платежа

6. **Отзывы пользователей:**
   - Оценка мероприятия (1-5)
   - Текстовый комментарий
   - Статус модерации

7. **Уведомления:**
   - Тип уведомления
   - Заголовок и текст сообщения
   - Связь с событием или билетом

8. **Сообщения обратной связи:**
   - Имя и email отправителя
   - Текст сообщения
   - Статус обработки

#### Выходные данные

Выходными данными системы являются:

1. **QR-коды билетов**: Уникальные QR-коды для каждого билета, содержащие информацию о событии, месте и пользователе

#### Связи между таблицами

База данных организована с использованием реляционной модели данных. Основные связи:

1. **Users ↔ Orders** (1:N) - Один пользователь может иметь множество заказов
2. **Users ↔ UserTickets** (1:N) - Один пользователь может купить множество билетов
3. **Users ↔ Payments** (1:N) - Один пользователь может совершить множество платежей
4. **Users ↔ Reviews** (1:N) - Один пользователь может оставить множество отзывов (но только один на событие)
5. **Users ↔ Notifications** (1:N) - Один пользователь может получить множество уведомлений
6. **Users ↔ Seats** (1:N, опционально) - Один пользователь может зарезервировать множество мест
7. **Events ↔ TicketTypes** (1:N) - Одно событие может иметь множество типов билетов
8. **Events ↔ Seats** (1:N) - Одно событие может иметь множество мест
9. **Events ↔ UserTickets** (1:N) - Одно событие может иметь множество проданных билетов
10. **Events ↔ Reviews** (1:N) - Одно событие может иметь множество отзывов
11. **Orders ↔ Payments** (1:N) - Один заказ может иметь множество платежей
12. **Seats ↔ UserTickets** (1:1) - Одно место связано с одним билетом

### 2.3. Описание программных средств

#### Серверная часть

**C# и .NET 8.0**

Для разработки серверной части приложения был использован язык программирования C#. C# — современный объектно-ориентированный язык программирования, разработанный компанией Microsoft. Данный язык поддерживает такие парадигмы программирования, как процедурное программирование, объектно-ориентированное программирование, обобщенное программирование, функциональное программирование, обеспечивает модульность, раздельную компиляцию, обработку исключений, абстракцию данных, объявление типов (классов) объектов, виртуальные функции. Стандартная библиотека включает, в том числе, общеупотребительные контейнеры и алгоритмы. C# сочетает свойства как высокоуровневых, так и низкоуровневых языков, что делает его идеальным выбором для разработки веб-приложений.

**ASP.NET Core MVC**

Серверная часть была реализована на ASP.NET Core MVC — мощном и современном фреймворке для построения веб-приложений и RESTful API. ASP.NET Core обеспечивает:
- **Высокую производительность**: Благодаря оптимизированному runtime и асинхронной обработке запросов
- **Кроссплатформенность**: Работа на Windows, Linux и macOS
- **Модульность**: Гибкая архитектура с поддержкой внедрения зависимостей (Dependency Injection)
- **Middleware-конвейер**: Возможность настройки обработки запросов на различных этапах
- **Удобную маршрутизацию**: Гибкая система маршрутизации запросов
- **Встроенную безопасность**: Поддержка аутентификации и авторизации

MVC-архитектура (Model-View-Controller) позволяет четко разделять логику приложения, бизнес-правила и представление данных, что улучшает поддерживаемость, тестируемость и масштабируемость кода.

**Entity Framework Core**

Для работы с базой данных используется Entity Framework Core — современный ORM (Object-Relational Mapping) фреймворк от Microsoft. EF Core обеспечивает:
- **Удобное взаимодействие с БД**: Работа с данными через объекты C# вместо прямых SQL-запросов
- **Миграции**: Автоматическое управление схемой базы данных через миграции
- **LINQ-запросы**: Использование Language Integrated Query для работы с данными
- **Трекинг изменений**: Автоматическое отслеживание изменений объектов
- **Поддержка транзакций**: Обеспечение целостности данных

**Microsoft SQL Server**

Для хранения данных была выбрана Microsoft SQL Server — мощная реляционная система управления базами данных. SQL Server обеспечивает:
- **Надежность**: Высокий уровень надежности и отказоустойчивости
- **Производительность**: Оптимизированный движок для обработки больших объемов данных

- **Масштабируемость**: Возможность масштабирования от небольших приложений до крупных корпоративных систем

- **Инструменты**: Богатый набор инструментов для управления и разработки (SQL Server Management Studio)

### 2.4. Описание физической структуры базы данных

- Адаптивная сетка для отображения карточек событий

**2. Страница события (Event)**
Назначение: Детальная информация о мероприятии и бронирование мест
Элементы интерфейса:
- Изображение события и основная информация (название, дата, место, цена)
- Описание мероприятия и состав исполнителей
- Интерактивная схема зала (SeatMap) с визуализацией мест
- Легенда статусов мест (доступно, зарезервировано, продано)
- Панель выбранных мест с общей стоимостью
- Кнопка оформления заказа
- Секция отзывов о мероприятии

**3. Страница профиля (Profile)**
Назначение: Управление личными данными и просмотр билетов
Элементы интерфейса:
- Вкладки: "Профиль", "Мои билеты", "Настройки"
- Форма редактирования личных данных
- Список активных и использованных билетов с QR-кодами
- История заказов
- Настройки уведомлений

**4. Страницы аутентификации (SignIn, SignUp)**
Назначение: Вход и регистрация пользователей
Элементы интерфейса:
- Форма входа с полями email и пароль
- Форма регистрации с валидацией полей
- Индикаторы валидации в реальном времени
- Сообщения об ошибках
- Ссылки на восстановление пароля

**5. Страницы информации (About, Contact, FAQ, Terms, Privacy, Cookies)**
Назначение: Предоставление информации о платформе и правовых документов
Элементы интерфейса:
- Структурированный текст с заголовками
- Форма обратной связи на странице Contact
- Аккордеон с вопросами и ответами на странице FAQ
- Навигационные ссылки в футере

#### Невизуальные элементы интерфейса

**Система состояний:**
- **Загрузка данных**: Индикаторы загрузки (Skeleton, Spinner) при получении данных с сервера
- **Ошибки**: Toast-уведомления для отображения ошибок с возможностью закрытия
- **Успешные операции**: Зеленые уведомления об успешном выполнении действий
- **Пустые состояния**: Сообщения при отсутствии данных (например, "У вас пока нет билетов")

**Управление состоянием:**
- **UserContext** — глобальное состояние пользователя (авторизован/не авторизован, данные профиля)
- **ThemeContext** — управление темой интерфейса (светлая/темная)
- **TanStack Query** — кэширование и синхронизация данных с сервером
- **React Router** — управление навигацией и состоянием URL

**Валидация форм:**
- Клиентская валидация с использованием библиотеки react-hook-form
- Визуальная индикация ошибок под полями ввода
- Проверка формата email, длины пароля, обязательных полей
- Асинхронная валидация (проверка доступности email при регистрации)

**Обработка ошибок:**
- Централизованная обработка ошибок API через interceptors Axios
- Отображение понятных сообщений об ошибках пользователю
- Логирование ошибок для отладки
- Fallback-компоненты при критических ошибках

#### Адаптивность и доступность

**Адаптивный дизайн:**
- Mobile-first подход к разработке
- Breakpoints: mobile (< 768px), tablet (768px - 1024px), desktop (> 1024px)
- Адаптивная сетка с использованием CSS Grid и Flexbox
- Оптимизация изображений для различных размеров экранов

**Доступность (A11y):**
- Семантическая HTML-разметка
- Поддержка навигации с клавиатуры
- ARIA-атрибуты для screen readers
- Достаточный контраст цветов для читаемости
- Альтернативный текст для изображений

#### Анимации и переходы

**Библиотека Framer Motion:**
- Плавные переходы между страницами
- Анимации появления элементов при прокрутке (scroll animations)
- Hover-эффекты на интерактивных элементах
- Анимации загрузки и состояний

**Принципы анимаций:**
- Минималистичные, ненавязчивые анимации
- Длительность анимаций не превышает 300-500ms
- Использование easing-функций для естественного движения
- Отключение анимаций для пользователей с предпочтением reduced motion

#### Цветовая схема и типографика

**Цветовая палитра:**
- Поддержка светлой и темной темы
- Основной цвет (Primary) для акцентов и кнопок действий
- Вторичный цвет (Secondary) для фонов и карточек
- Цвета состояний: success (зеленый), error (красный), warning (желтый), info (синий)

**Типографика:**
- Система размеров шрифтов: от 12px (small) до 48px (heading-xl)
- Использование системных шрифтов для быстрой загрузки
- Иерархия заголовков для структурирования контента
- Оптимальная высота строки для читаемости

## ЗАКЛЮЧЕНИЕ

В ходе выполнения курсового проекта была разработана «Платформа для организации мероприятий» — современное веб-приложение, предназначенное для автоматизации процессов организации мероприятий, бронирования билетов, управления заказами и платежами, а также взаимодействия с пользователями. Проект успешно реализован с использованием актуальных технологий, включая C#, ASP.NET Core MVC, React.js и Microsoft SQL Server, что обеспечило высокую производительность, надежность, удобство использования и масштабируемость системы.

**Реализован полный функционал для всех групп пользователей:**

1. **Посетители:**

   - Генерация QR-кодов для билетов
   - Оставление отзывов о мероприятиях
   - Получение уведомлений о новых событиях
